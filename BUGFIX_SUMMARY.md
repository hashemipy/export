# خلاصه حل سه مسئله Stock Sync

## ✅ مسئله ۱: فقط موجودی کاهشی sync می‌شود

### مشکل
- وقتی سفارش کنسل می‌شود و موجودی افزایش می‌یابد، hook فراخوانی نمی‌شود
- تنها تغییرات کاهشی (reduce_order_stock) sync می‌شدند

### حل
**فایل:** `includes/StockSync.php` (خط 78-81)
```php
// اضافه کردن hook برای تمام تغییرات موجودی
add_action('woocommerce_product_stock_set_value', [$this, 'on_stock_changed'], 10, 1);
```

### نتیجه
- اکنون **تمام** تغییرات موجودی (افزایشی + کاهشی) sync می‌شوند
- کنسل شدن سفارش باعث بازگشت موجودی به سایت دیگر می‌شود

## ✅ مسئله ۲: تنظیم جهت sync (یک‌طرفه یا دوطرفه)

### مشکل
- کاربر نمی‌تواند انتخاب کند که موجودی در کدام جهت sync شود
- همیشه دوطرفه بود (هارد‌کد)

### حل

#### 1️⃣ **Settings.php - تنظیم جدید** (خط 180-188)
```php
'sync_direction' => sanitize_text_field($_POST['sync_direction'] ?? 'bidirectional')
```

#### 2️⃣ **Settings.php - فیلد UI** (خط 397-445)
اضافه کردن dropdown با سه گزینه:
- **دوطرفه** (پیش‌فرض): موجودی از هر دو سایت اعمال می‌شود
- **فقط سایت ۱ → ۲**: تنها از سایت ۱ به ۲
- **فقط سایت ۲ → ۱**: تنها از سایت ۲ به ۱

#### 3️⃣ **Settings.php - get_config defaults** (خط 752-754)
```php
'sync_direction' => 'bidirectional'
```

#### 4️⃣ **StockSync.php - فیلتر جهت** (خط 299-308)
```php
// بررسی جهت sync - اگر این جهت فعال نیست، skip کن
$sync_direction = $config['sync_direction'] ?? 'bidirectional';
if ($sync_direction === 's1_to_s2' && $direction === 's2_to_s1') {
    $this->logging->log('sync', 'debug', "Sync مسدود: تنظیمات فقط یک‌طرفه");
    return;
}
```

### نتیجه
- ادمین می‌تواند sync را یک‌طرفه یا دوطرفه کنترل کند
- تنظیمات در پنل مدیریت قابل تغییر است

## ✅ مسئله ۳: Duplicate Mapping در آپلود دستی

### مشکل
- هنگام آپلود دستی محصول، mapping دو بار ثبت می‌شود
- اگر درخواست retry شود، mapping تکراری ایجاد می‌شود
- جدول `pie_stock_map` دارای ردیف‌های تکراری می‌شود

### حل

#### 1️⃣ **StockSync.php - idempotent register_mapping** (خط 826-848)
```php
// بررسی دقیق - آیا این mapping دقیقاً موجود است؟
$existing = $wpdb->get_row($wpdb->prepare(
    "SELECT id FROM {$table} 
     WHERE s1_product_id = %d AND s2_product_id = %d
     AND (s1_variation_id <=> %s) AND (s2_variation_id <=> %s)",
    $s1_product_id, $s2_product_id, $s1_variation_id, $s2_variation_id
));

if ($existing) {
    // بروز رسانی فقط (duplicate نه)
    $wpdb->update($table, [...], ['id' => $existing->id]);
    return $existing->id;
}
```

#### 2️⃣ **Transfer.php - idempotent check** (خط 147-200)
- قبل از `register_mapping` فراخوانی، بررسی می‌کنیم آیا mapping وجود دارد
- اگر وجود داشت: فقط نام/ویژگی‌ها بروز می‌شود
- اگر جدید بود: ثبت می‌شود

### نتیجه
- **Duplicate mapping حذف شد**
- بسیار ایمن و idempotent
- اگر دستی محصول دو بار آپلود شود، فقط یک mapping ایجاد می‌شود

```
1. به سایت ۱ برو
2. یک سفارش ایجاد کن (موجودی کاهش می‌یابد)

```

```

2. در سایت ۲ موجودی تغییر بده
3. موجودی سایت ۱ باید تغییر نکند ✅
4. در سایت ۱ موجودی تغییر بده
5. موجودی سایت ۲ باید تغییر کند ✅
```

```
1. یک محصول دانلود کن (JSON)

4. فقط یک ردیف mapping باید وجود داشته باشد ✅
5. دوباره آپلود کن
6. هنوز فقط یک ردیف ✅
```
