## ساختار JSON صحیح

### محصول متغیر:
```json
{
  "type": "variable",
  "attributes": [
    {
      "name": "رنگ",
      "options": ["سرمه ای", "طوسی", "قهوه ای", "یشمی"]
    },
    {
      "name": "سایز",
      "options": ["40", "45", "50", "55", "60", "65"]
    }
  ]
}
```

### متغیرها:
- **تعداد متغیرها = تعداد رنگ‌ها × تعداد سایز‌ها**
- برای محصول ما: 4 رنگ × 6 سایز = **24 متغیر**
- هر متغیر باید **SKU منحصر به فرد** داشته باشد

---

## مثال کامل (محصول شما)

```
محصول: تیشرت شلوارک کارنیوال
SKU: 1521
قیمت: 865,000 تومان
موجودی: 24 عدد (1 عدد برای هر متغیر)

ویژگی‌ها:
- رنگ: سرمه ای، طوسی، قهوه ای، یشمی (4 گزینه)
- سایز: 40، 45، 50، 55، 60، 65 (6 گزینه)

متغیرها:
1. 1521-DARK-BLUE-40
2. 1521-DARK-BLUE-45
... (کل 24 متغیر)
```

---

## خطاهای رایج

### ❌ SKU تکراری
```
خطا: همه متغیرها SKU یکسان دارند
✓ حل: هر متغیر باید SKU منحصر به فرد داشته باشد
مثال: 1521-DARK-40, 1521-DARK-45, ...
```

### ❌ متغیرهای ناقص
```
خطا: فقط 4 متغیر برای 4 رنگ و 6 سایز
✓ حل: باید 24 متغیر باشد (4 × 6)
```

### ❌ ویژگی‌ها وجود ندارند
```
خطا: محصول ساده به جای متغیر ایجاد می‌شود
✓ حل: حتماً "attributes" را درج کنید
```

## نکات اضافی

- فایل JSON باید **معتبر** باشد (می‌تونید در jsonlint.com بررسی کنید)
- آدرس سایت مقصد باید **صحیح** باشد (شامل https://)
- برای اپلود‌های بزرگ، ممکن است **کمی زمان** ببرد
- اگر **خطا** دید، پیام خطا را بخوانید و اصلاح کنید
