# سیستم انتقال محصولات متغیر

سلام! این پروژه یک **سیستم کامل و حرفه‌ای** برای انتقال محصولات متغیر (Variable Products) بین دو سایت است.

## مشکل اصلی که حل می‌کند

کاربران اغلب این مشکلات را دارند:

❌ **مشکل اول:** محصول متغیر دانلود می‌شود اما بدون ویژگی‌ها و متغیرها

❌ **مشکل دوم:** ویژگی‌ها (رنگ، سایز) در سایت مقصد ایجاد نمی‌شوند

❌ **مشکل سوم:** محصول ساده ایجاد می‌شود نه متغیر

❌ **مشکل چهارم:** موجودی محصول صحیح نیست

## حل ما

```
📥 دانلود هوشمند     🔧 پردازش تکیه‌داری    📤 اپلود ترتیبی
├─ محصولات         ├─ تنظیف داده‌ها      ├─ دسته‌بندی‌ها ۱
├─ دسته‌بندی‌ها      ├─ بررسی یکپارچگی     ├─ ویژگی‌ها ۲
├─ ویژگی‌ها         ├─ حذف تکراری‌ها      ├─ محصول ساده ۳
└─ تمام وابستگی‌ها   └─ محاسبه موجودی     └─ متغیرها ۴
```

## نحوه کار

### گام ۱: دانلود

دنبال کنید:
```typescript
const downloader = new ProductDownloader('https://site-mabda.com');
const data = await downloader.downloadComplete();
// ✓ تمام محصولات + ویژگی‌ها + دسته‌بندی‌ها
```

**آنچه استخراج می‌شود:**
- محصولات متغیر (همه فیلدها)
- دسته‌بندی‌های مادر و فرزند
- ویژگی‌ها و تمام مقادیرشان
- تصاویر محصولات
- موجودی هر متغیر

### گام ۲: پردازش

تنظیف و بررسی:
```typescript
const processor = new DataProcessor();
const { data, errors } = processor.processAll(
  products,
  categories,
  attributes
);
// ✓ تمام داده‌ها صحیح و معتبر
```

**چه کاری انجام می‌شود:**
- ✓ حذف SKU تکراری
- ✓ تکراری‌های ویژگی حذف می‌شود
- ✓ دسته‌های مادر قبل از فرزند
- ✓ موجودی کل محاسبه می‌شود
- ✓ خطاهای ممکن شناسایی می‌شود

### گام ۳: اپلود

اپلود به ترتیب صحیح:
```typescript
const uploader = new ProductUploader('https://site-maqsad.com');
const result = await uploader.uploadComplete(
  categories,
  attributes,
  products
);
```

**ترتیب اپلود:**
```
۱. دسته‌بندی‌ها (مادر ابتدا)
   ↓
۲. ویژگی‌ها (رنگ، سایز، و...)
   ↓
۳. محصول اصلی (بدون متغیر)
   ↓
۴. متغیرها (هر combination)
```

## استفاده سریع

### برای انتقال کامل

```typescript
import { ProductMigrationManager } from '@/lib/plugins';

const manager = new ProductMigrationManager(
  'https://source.com',
  'https://destination.com',
  'your-api-key'
);

const result = await manager.migrateComplete();

console.log(`✓ ${result.uploadResult.createdProducts} محصول ایجاد شد`);
console.log(`✓ ${result.uploadResult.createdAttributes} ویژگی ایجاد شد`);
console.log(`✓ ${result.uploadResult.createdCategories} دسته‌بندی ایجاد شد`);
```

### فقط دانلود

```typescript
const downloadedData = await manager.downloadOnly();
const fileBlob = await downloader.downloadFile(downloadedData);

// ذخیره فایل
const link = document.createElement('a');
link.href = URL.createObjectURL(fileBlob);
link.download = 'products-export.json';
link.click();
```

### اپلود فایل موجود

```typescript
// فایل قبلاً دانلود شده را اپلود کنید
const fileContent = await file.text();
const jsonData = JSON.parse(fileContent);

const result = await manager.uploadOnly(jsonData);
```

## ویژگی‌های اصلی

✅ **دانلود ذکی**
- استخراج خودکار وابستگی‌ها
- حفظ ارتباطات مادر-فرزند
- حذف تکراری‌ها

✅ **پردازش توانمند**
- بررسی یکپارچگی
- تنظیف داده‌ها
- لاگ دقیق

✅ **اپلود ترتیبی**
- ترتیب صحیح ایجاد
- موجودی دقیق
- بررسی خطا

✅ **UI کاربر‌پسند**
- رابط گرافیکی ساده
- نمایش وضعیت زنده
- دانلود/اپلود سهل

## نمونه عملی

### سناریو: انتقال محصول "تیشرت کارنیوال"

**مشخصات محصول:**
- 4 رنگ: سرمه‌ای، طوسی، قهوه‌ای، یشمی
- 6 سایز: 40، 45، 50، 55، 60، 65
- 10 متغیر (نه 24 تا)
- موجودی: 35 عدد

**فرایند:**

```
۱. دانلود از سایت مبدا
   ✓ محصول + تمام ویژگی‌ها
   ✓ دسته‌بندی: "آستین کوتاه" → "بچگانه"

۲. پردازش
   ✓ بررسی: همه 10 متغیر معتبر
   ✓ SKU: 1521-60-DARK_BLUE, 1521-65-DARK_BLUE, ...
   ✓ موجودی کل: 5+3+2+2+10+8+4+6+5+12 = 57 ❌
   ⚠️  هشدار: موجودی مطابقت ندارد

۳. اپلود به سایت مقصد
   ✓ ایجاد دسته‌بندی "آستین کوتاه" (parent)
   ✓ ایجاد دسته‌بندی "بچگانه" (child)
   ✓ ایجاد ویژگی "رنگ" با 4 مقدار
   ✓ ایجاد ویژگی "سایز" با 6 مقدار
   ✓ ایجاد محصول "تیشرت کارنیوال"
   ✓ ایجاد 10 متغیر

۴. نتیجه
   ✅ سایت مقصد حالا محصول متغیر کامل دارد
   ✅ تمام ویژگی‌ها صحیح
   ✅ تمام متغیرها موجود
```

## سوالات متداول

**س: آیا می‌توام فقط بعضی محصولات را دانلود کنم؟**
ج: بله، فیلترهای سایت مبدا را استفاده کنید

**س: اگر اپلود ناموفق بود چه؟**
ج: فایل JSON ذخیره شده را دوباره اپلود کنید

**س: آیا ویژگی‌های قدیم حذف می‌شوند؟**
ج: خیر، فقط نو‌ها اضافه می‌شوند

**س: موجودی کل چگونه محاسبه می‌شود؟**
ج: جمع موجودی تمام متغیرها

**س: آیا تصاویر کپی می‌شوند؟**
ج: بله، اگر URL ها معتبر باشند
